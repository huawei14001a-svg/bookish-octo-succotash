# VRF Exchange Bot

Telegram-биржа токена **#VRF** (покупка/продажа/стейкинг) + отдельный
токен **#SCAM** с переводами по текстовой команде `пер` и генерацией
карточки-картинки. Плюс публичный REST API для сторонних разработчиков.

## Структура репозитория

```
.
├── vrf_exchange_bot.py       ← основной файл бота (запускается напрямую)
├── scam_assets_data.py       ← фон + шрифты для карточки SCAM, в base64
├── requirements.txt          ← зависимости для pip
├── Procfile                  ← команда запуска для Railway
├── API_DOCS.md               ← документация REST API для разработчиков
├── api_docs.html             ← та же документация в виде HTML-страницы
└── assets/                   ← НЕОБЯЗАТЕЛЬНО, только запасной вариант
    ├── scam_bg.png
    └── fonts/
        ├── DejaVuSans-Bold.ttf
        └── DejaVuSans.ttf
```

### Почему `scam_assets_data.py`, а не просто файлы в `assets/`

Фон и шрифты для карточки перевода SCAM закодированы в base64 и лежат
**прямо в python-файле** `scam_assets_data.py`, а не как отдельные
бинарники в папке `assets/`. Это сделано намеренно: отдельные
картинки/шрифты слишком легко потерять при деплое (не докоммитили,
`.gitignore` их исключил, платформа скопировала не всё и т.д.).
`.py`-файл в этом смысле надёжнее — если деплоится сам бот, деплоится
и он.

**Обе части обязательно должны лежать рядом, в корне репозитория:**
`vrf_exchange_bot.py` и `scam_assets_data.py`.

Папка `assets/` при этом оставлена как необязательный запасной
источник — если по какой-то причине `scam_assets_data.py` не
импортируется, бот попробует прочитать файлы оттуда, а если и там
пусто — сгенерирует похожую по стилю карточку программно, но уже без
твоего оригинального фото.

## Самодиагностика

Команда **`/checkassets`** (только для админов из `ADMIN_IDS`) прямо в
Telegram покажет, что именно бот видит на сервере:

```
📂 Скрипт запущен из: /app
✅ scam_assets_data.py: найден и импортирован
✅ embedded scam_bg: 1564 КБ декодировано
✅ embedded bold-шрифт: 692 КБ декодировано
ℹ️ (запасной вариант) папка assets/: не найдена — /app/assets
🖼 Итог: фон будет взят из scam_assets_data.py (основной источник)
```

То же самое пишется в лог при каждом старте — ищи блок
`=== SCAM assets diagnostics ===` в Railway → Logs сразу после деплоя.

## Переменные окружения (Railway → Variables)

| Переменная | Обязательна | Описание |
|---|---|---|
| `BOT_TOKEN` | ✅ | Токен бота от @BotFather |
| `DB_PATH` | ❌ | Путь к SQLite-файлу (по умолчанию `vrf_exchange.db`) |
| `ADMIN_IDS` | ❌ | ID администраторов через запятую |
| `PORT` | ❌ | Порт для REST API — Railway подставляет сам |
| `API_CORS_ORIGIN` | ❌ | CORS-домен для API (по умолчанию `*`) |

## Деплой на Railway

1. Создай новый репозиторий, залей туда все файлы из этого пакета
   (структуру папок для `assets/` соблюдать не обязательно — она
   опциональна, но `scam_assets_data.py` должен лежать рядом с
   `vrf_exchange_bot.py`)
2. Railway → New Project → Deploy from GitHub repo
3. Settings → Variables → добавь `BOT_TOKEN`
4. Settings → Networking → Public Networking → Target Port `8080`
   (нужно для REST API; сам бот работает через long polling и в этом
   не нуждается)
5. Дождись деплоя, проверь `/health` на выданном домене — должен
   вернуть `{"ok": true}`
6. В Telegram у **@BotFather** → `/mybots` → твой бот → **Bot Settings**
   → **Group Privacy** → **Turn off** (иначе бот не увидит команду
   `пер` в группах — увидит только сообщения с `/`)
7. Удали и заново добавь бота в группу, если он там уже был —
   изменение Group Privacy не применяется задним числом
8. Напиши боту `/checkassets` — убедись, что фон берётся из
   `scam_assets_data.py`

## Быстрая проверка после деплоя

```
/start         — регистрация, приветствие
/market        — текущая цена VRF
/balance       — баланс USD/VRF/SCAM
/checkassets   — диагностика фона/шрифтов (админ)
пер 0,048      — перевод SCAM (ответом на чьё-то сообщение в группе)
```

## Основные команды

**Биржа:** `/market` `/price` `/chart` `/buy` `/sell`
**Стейкинг:** `/stake` `/stakes`
**Токен SCAM:** `пер <сумма>` (ответом на сообщение получателя)
**Аккаунт:** `/balance` `/apikey` `/apikeys`
**Админ:** `/admin` `/checkassets` `/setprice` `/addusd` `/addvrf` `/addscam`

Полный список — `/help` в самом боте.
